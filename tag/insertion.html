插入排序和Shell排序

二 15 四月 2014
By Rong Chen

In Algorithm.

tags: sortshellinsertion

插入排序的算法在打扑克牌时经常使用，算法比较简单，但是它和冒泡法一样，涉及相邻元素的交换，我们在评价算法效率时往往只看到循环，而忽略循环里的内容，今天这篇文章就说这个。

插入排序的算法如下：

void insertion_sort(int b[], int n) {
    for (int i = 1; i < n; i ++) {
        for (int j = i; j > 0 && (b[j] < b[j-1]); j --) {
            exch(b, j, j-1);
        }
    }
}

试想如果数组中最后一个元素就是最小值，那么它要移动n-1次才能到达正确的位置。

Shell排序是基于插入排序开发出来的，名字来源于作者，就是为了解决插入排序中类似上述情况中移动次数过多的问题，算法比插入排序复杂多了，代码如下：

void shell_sort(int b[], int n) {
    int h = 1;
    while (h < n/3) h = h * 3 + 1;
    while (h >= 1 ) {
      for (int i = h; i < n; i ++) {
        for (int j = i; j >= h && b[j] < b[j-h]; j -= h)
          exch(b, j, j-h);
      }
      h /= 3;
    }
}

可以看到，Shell排序经过了多次插入排序，但是并不是对相邻元素进行排序，它排序的元素之间索引间隔为h，最终h会变成1，这时就变成了纯粹的插入排序，即1-sort。

写到这里可以看出，Shell排序比插入排序要对待排序数组进行多次处理，最后一次才是真正的1-sort插入排序，即才等价于insertion_sort的代码。前面处理的目的只是将数组进行笼统的排序，就算最小值位于数组末端也没有问题，通过h-sort跳跃式地移动到数组前端。

比较一下上面两段代码的效率吧：

int main(int c, char **v) {
    int a[50000];
    int n = sizeof(a)/sizeof(int);
    for (int i = 0; i < n; i ++) {
        a[i] = rand() % 10000;
    }
    xxxx_sort(a, n);
}

结果如下：

time insertion_sort
real    0m4.991s
user    0m4.986s
sys 0m0.004s

而：

time shell_sort
real    0m0.104s
user    0m0.035s
sys 0m0.004s

50倍！！
